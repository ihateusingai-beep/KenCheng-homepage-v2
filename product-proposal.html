Dimension 1 · New Feature Development

新功能開發

基於現有 Astro 7 + Tailwind v4 架構，結合市場上導航站嘅最佳實踐，以下六個功能按優先序排列，全部可在無後端、無數據庫嘅條件下實現（localStorage + GitHub Actions）。

⌘
⌘K 全站快速搜尋面板
用 Cmd/Ctrl+K 喚起 fuzzy search，跨 category / tag / name 即時過濾，keyboard-only 導航（↑↓ Enter Esc）。對比：v1 係 input focus + dropdown，呢個係標準 command palette design，學習成本接近零。
預期效果：用戶留存 +25%，工具發現時間 -60%。教師喺課堂上即時搵到需要嘅工具。
P0 — 立即實作 localStorage client-side
✓
Site Health 狀態徽章
每個工具 card 顯示 alive / slow / dead / redirect 狀態。Dead link nightly check 由 GitHub Actions cron job 驅動（`A1` Roadmap item），每 24 小時爬取所有 URL，更新 health 欄位後 commit 回庫。學生、老師唔再撞 dead link。
技術路徑：GitHub Actions + curl head request → 更新 frontmatter → `git add + commit` → push。Service Worker cache version 同步 bump。
P1 — v2 Roadmap F2 GitHub Actions CI/CD
★
快速收藏書籤（Quick Add）
任意頁面一鍵 bookmark 到 localStorage，支援自訂名稱 + tag。唔需要 auth，唔需要後端。bookmark list 頁面展示全部已收藏工具，可匯出為 JSON 或複製為 Markdown list。與現有 v1 `bookmark-sync.html` 功能整合升級。
P1 — v2 Roadmap F3 localStorage no-backend
📁
書籤資料夾分組（F4）
用戶自建 folder（e.g.「中文備課」「STEM 課堂」），將書籤拖入不同 folder。folder 之間可移動工具、排序。完全 client-side，資料存 localStorage，無 server-side 需求。
P2 — v2 Roadmap F4 localStorage drag-drop
📊
使用統計儀表板（Own Analytics）
追蹤：熱門工縣（bookmark count）、category 分布、平均停留時間。所有數據存在 localStorage（學生端），可選上報到 localStorage（老師端 aggregated view）。無 Google Analytics，GDPR-compliant，完全 self-contained。
P2 — 新功能 localStorage GDPR-safe
🤖
AI 工具推薦引擎升級
v1 已有 `__recommendTools` diversity-aware engine（role + cat coverage）。v2 升級：根據 bookmark history 實時推薦（localStorage inference，唔需要 server）、每週精選「本週新工具」（基於 frontmatter `date_added` 排序）、支援「隨機一個工具」按鈕（適用於：老師唔知用邊個工具時）。

Technical Risk Assessment

技術風險評估

每個功能嘅潛在風險分為四級（Critical / High / Medium / Low），包含舒緩對策。基於 Astro 7 + Tailwind v4 + GitHub Pages 呢個特定 stack。

Critical
GitHub Actions Cron 配額
GitHub Free plan cron job 每月 2000 分鐘。Dead link check 每日凌晨跑 63 個 URL（每次 ~30 秒）= ~30 分鐘/月，喺限額內。但若加 tool count 到 300+，會逼近限額。
✅ mitigation：加 concurrency limit、cache previous results、用 `HEAD` 而非 `GET` 减少時間。
High
localStorage 容量限制
localStorage 上限 ~5MB（瀏覽器）。Bookmark + folder + analytics 數據若超過 5MB，iOS Safari 會静默fail write，數據丢失但無錯誤提示。
✅ mitigation：書籤 JSON 超过 3MB 時 warning user；優先存必要欄位（url + title + folder），analytics 數據探樣而非全量。
High
⌘K Palette 與 Screen Reader 衝突
Command palette 係高度客製化 widget，若 skip-link / aria-live 实现不完整，會干擾 screen reader 用戶嘅 navigation flow。
✅ mitigation：palette 實現严格 ARIA dialog spec；上綫前用 NVDA / VoiceOver 測試；提供 keyboard-only fallback。
Medium
Service Worker Cache 版本同步
每次 deploy 需要 bump SW cache version，否則舊 SW 會 stale cache 回朔舊頁面。呢個係 v1 已知問題，v2 需要確保 `astro.config.mjs` build script 自動 bump SW version。
✅ mitigation：Astro build hook 自動 inject timestamp 或 hash 到 sw.js；每次 build 時校驗 sw.js 與 manifest.json version 一致。
Medium
Lighthouse CI False Positive
Lighthouse CI threshold 設得太高（e.g. Performance ≥ 95）會造成 flaky failure，尤其係 shared runner 網絡波動造成 TTFB 飆升。
✅ mitigation：threshold 設合理值（Performance ≥ 85, Accessibility ≥ 90）；加入 retry 機制（flaky 最多 2 次重試）；將 Lighthouse 結果 trend 化，唔設 hard gate。
Low
OG Image Gen 依賴外部服務
若 OG image gen 用 Satori（Vercel edge），生成服務 downtime 會造成 build fail。
✅ mitigation：用 Astro endpoint generate SVG → 轉 PNG，全部 client-side 完成，無外部依賴。或 fallback 為 static fallback OG image。

Implementation Roadmap · Suggested Phasing

建議實施路線圖

基於 v2 現有 roadmap，融合上述四個維度嘅優化項目。Sprint 以 2 週為單位。

  1. Sprint 1 · Foundation
    ⌘K Palette + Dead Link Check
    同時實作 F1（⌘K）和 A1（dead link nightly）。兩者技術上無依賴，但都係「基礎設施級」功能，其他功能依賴佢哋。⌘K 完成後立即降低所有用戶嘅 tool discovery friction。
    F1 ⌘K A1 Dead Link Critical Path
  2. Sprint 2 · Bookmarks
    Quick Add Bookmark + Bookmark Folders
    F3 + F4 同步開發（localStorage schema 一次定義）。Bookmark import/export JSON v2。整合現有 v1 `bookmark-sync.html` 用戶嘅 localStorage data（backward compat）。
    F3 Quick Add F4 Folders localStorage schema
  3. Sprint 3 · Infrastructure
    CI Smoke Gate + Sitemap/RSS
    A6（Lighthouse CI + 5 invariants）鋪好 quality gate 後，A4（sitemap + RSS）無痛上綫。呢個 sprint 主要係 reliability + SEO 基建，为後續功能守底綫。
    A6 CI Gate A4 Sitemap/RSS A2 OG Image
  4. Sprint 4 · UX Polish
    URL State + Theme System + Touch Target
    URL query param 重構現有 filter system（breaking change，需要 careful QA）。Dark/Light/System theme 實作 + 現有 v1 Gundam theme 移植。Mobile touch target 合規修復。
    URL state Theme system Mobile a11y
  5. Sprint 5 · AI + Analytics
    AI Recommendation Engine + Own Analytics
    F6 升級 `__recommendTools` engine 加 bookmark-based inference + 每週精選 + 隨機工具。F5 使用統計儀表板（全部 client-side，GDPR-safe）。
    F5 Analytics F6 AI engine P2
  6. Sprint 6+ · Polish
    WCAG Audit + Empty States + PWA Deep Integration
    WCAG 2.1 AA full audit + fix。所有 empty state / zero results 場景設計。Share API、PWA deep integration（home screen shortcut、offline indicator）。呢個 sprint 可以持續，視用戶 feedback 動態調整優先序。
    WCAG AA Empty states PWA deep
