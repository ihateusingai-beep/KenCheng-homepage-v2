01 — 新功能開發
具市場競爭力的新功能
解決用家痛點、提升黏著度、差異化競爭
🔮 個人化推薦引擎
移植 v1 的 __recommendTools 多樣性算法，按 bookmark、瀏覽歷史、 角色（老師/學生）動態推薦工具。支援「為你精選」板塊，提升回訪率。
P0 · 高impact
⭐ Bookmark + 文件夾
localStorage 書籤系統，用家可收藏工具、建立自訂文件夾（教學/備課/學生）， 支援雲端同步（Future: 同學/同事共享）。一鍵導出書籤列表。
P1
🩺 工具健康監測
每週 GitHub Action 自動檢測 63 個 URL 可達性，結果寫入 frontmatter health 欄位。dead link → 🔴 下架，slow → 🟠 標記， alive → 🟢 確認。保持工具列表可信度。
基礎設施
📋 工具提交系統
用家可提交新工具 → pending review → 管理員審批 → 自動新增到 Content Collection。減少維護成本，社區驅動內容增長。
P2 · 長期
👥 學生專區（遷移）
v1 的 /students.html — 專為學生設計的精簡工具入口， 去除老師視角的功能（SEN 評分、教案），只留學生適用的工俱。 降低認知負擔，提升學生自主學習體驗。
P1
📱 QR Code 批量下載
v1 的 /download-qr.html — 工具二維碼批量生成 + 一鍵打包 ZIP。用家可為課堂電腦貼上 QR code，學生即掃即用。 配合 Bookmark 系統使用。
P2
🚀 收藏夾 Sync（未來）
登入後書籤自動雲端同步（Cloudflare D1 或 GitHub Gist）。 可跨設備、跨瀏覽器使用。同一老師的多台設備無縫銜接。
P2 · 未來

💡 市場差異化分析

目前市面上少有針對香港/SEN教師群體的精選工具導航。大部分 aggregator（Product Hunt、AllThings）唔够精準； 教育平台（ClassTools、Twinkl）則覆蓋範圍太廣、缺乏 AI 工具深度。 本產品的護城河在於：Ken 本人的實戰揀選（非機器人爬取）+ AI 應用敏銳度 + SEN 特殊場景適配。建議以「老師的私人AI助手工具箱」作為核心定位。

02 — UX 優化
使用者體驗改善
降低學習成本、提升操作效率、減少摩擦點
🎯 Onboarding Wizard
首次訪問：3-step 嚮導（老師/學生 → 科目 → 偏好）。 localStorage 記住身份，自動套用預設 Filter + 推薦。 返回用家唔再睇到空白主頁，提升首日價值感知。
P1
⌨️ 鍵盤捷徑覆蓋圖
? 鍵彈出快捷鍵覆蓋圖（⌘K、/、Esc、↑↓）。 新用家知道有 keyboard-driven 操作，降低滑鼠依賴。 配合 focus-visible 樣式，視覺障礙用家都適用。
P1
📤 Bookmark 導出 / 分享
一鍵導出書籤為 Markdown 列表（可用於週報、備課筆記）。 支援生成分享連結（encoded URL param），同事可查看同一份書籤。
P2
📖 工具詳情 Modal
點擊 card 彈出 modal，顯示完整 description、screenshot（可選）、 使用場景、替代方案。用家唔使離開頁面就可以評估工具。
P1
🔍 Search 結果高亮 + 分頁
⌘K palette 目前顯示 top-10。用 ⌘K 結果分頁（最多 50 條）， 加入 category facet filter（只看 AI / 數學），提升長尾搜索效率。
P2

📊 現有 UX 痛點分析

基於 audit 結果，以下係現有最大 UX gap：

  • 無 onboarding — 新用家第一眼睇到全部 63 個工具，無方向感。
  • ⌘K palette 只在首頁存在 — category 頁無 search，導致跳轉後體驗断層。
  • 工具 description 分散 — body text 而非 frontmatter，搜尋時無法 highlight。
  • URL state 未與 Filter bar 同步 — 篩選結果無法分享（filter.ts 已實現，需驗證）。
  • 無鍵盤捷徑提示 — ⌘K 存在但用家未必發現。
03 — UI 設計
視覺設計現代化
美觀直觀，符合現代設計趨勢，強化品牌形象
🌓 亮色模式深度優化
目前亮色模式的 aurora 動畫、card 樣式在淺色背景上視覺衝突。 建議：亮色模式停用 aurora，改為柔和渐变背景； card 改為白色 + 微陰影，完全適配 macOS/iOS 淺色主題。
P1
🖼️ OG Image + Social Preview
每個 category 頁面自動生成 OG image（標題 + icon + category 色調）。 現在所有 OG meta 只係 static text，social share 視覺效果差。 建議用 @resvg/resvg-js 或 Puppeteer (CI) 生成 OG 圖。
P2
📲 PWA Install Banner
manifest.json 已存在但無 install banner。用 beforeinstallprompt 事件 在適當時機（用家 bookmark 了 3+ 工具後）提示「安裝 App 版」， 提升回訪率與 engagement。配合 iOS Add-to-Homescreen 提示。
P1
⚡ 性能：Aurora 動畫優化
目前 aurora 動畫（3 層 radial-gradient + grid-drift）喺低端設備上可能造成 jank。 建議：prefers-reduced-motion 時完全停用 aurora； 使用 will-change: transform + content-visibility: visible 減少 repaint；考慮喺 1s 後降低動畫 frame rate。

04 — 自動化
智慧化自動流程
減少人工維護成本，確保數據時效性
🤖 Nightly Link Health Check（GitHub Action）
每晚 UTC 00:00 跑 GitHub Action：
1. 讀取 src/content/sites/*.md 所有 URL
2. 並發 HEAD request（max 10 parallel），超時 10s
3. 分類結果：200→alive、301/302→redirect、timeout/5xx→dead、>5s→slow
4. 自動 commit health: + last_checked: 更新
5. 若有新 dead link → Telegram / Email alert

價值：63 個 URL 中目前相當部分 health=unverified， 此 automation 可確保列表可信度，減少用家投訴「click 咗係 dead link」。
基礎設施 · P0
🧪 CI Smoke Test Gate
每次 PR push 觸發 CI：
1. astro build — 確認 0 error
2. 5 invariants check（見測試清單）
3. lighthouse CLI — performance ≥ 85、accessibility ≥ 90
4. HTML validate（無 missing alt、duplicate id）
5. Check broken internal links（lychee CI tool）

價值：防止 regression、確保每次 deploy 質量可控。
基礎設施

🔗 現有自動化缺口

  • 無 RSS feed — /rss.xml link 存在但 endpoint 404。@astrojs/rss dependency 已裝，祇欠 endpoint。
  • 無 sitemap — /sitemap-index.xml link 存在但 404。需要 @astrojs/sitemap integration。
  • SW version 未 sync — manifest.json 無 version 欄位，SW cache name 係 hardcoded kc-toolhub-v1。每次 deploy 應該 bump version。
  • Content frontmatter 無自動格式化 — date_added、order 係手動填，未有 CI validate required fields。
